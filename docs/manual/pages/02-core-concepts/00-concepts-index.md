# Core Concepts

**Purpose:** Give you the product model before the first real task gets noisy.

HappyRanch is easiest to understand as an operating system for a small AI-agent
org. You do not need to learn every page before starting. You do need these six
ideas:

1. **Tasks** are work to be executed.
2. **Threads** are conversations and coordination.
3. **Agents and teams** decide who does the work.
4. **Jobs and escalations** are the safety controls when work needs background
   execution or founder approval.
5. **Memory, KB, and Artifacts** are three different stores; they are not
   interchangeable.
6. **Usage** shows token consumption, not dollars.
